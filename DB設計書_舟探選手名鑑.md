# 舟☆探 選手名鑑 DB設計書（v1.0）

作成日: 2026-07-06
方針: **CSV2枚だけで運用する。** スプレッドシートで開けて、Claude Codeがそのままサイト生成に使える形。

---

## 全体像

```
[racers.csv]  選手そのものの情報（1選手＝1行）
     │
     │ 登録番号で紐づく
     ▼
[relations.csv] 人間関係の情報（1関係＝1行）
```

選手の情報と「関係」の情報を分けるのが最大のポイント。
関係を選手の行に書き込むと（例:「配偶者」列を作る）、兄弟が3人いる選手や弟子が7人いる選手で破綻する。関係は別の紙に1件ずつ書く、と覚えればOK。

## ② relations.csv（関係マスタ）

| 列名 | 内容 | 例 | ルール |
|---|---|---|---|
| id | 通し番号 | R0001 | 修正・削除の目印 |
| from_toban | 主の選手 | 4320 | |
| rel_type | 関係タイプ | 師匠 | 下の**関係タイプ辞書**から選ぶ |
| to_toban | 相手の登録番号 | 3941 | 選手でない場合は空欄 |
| to_name | 相手の名前 | 山田太郎 | 相手が選手ならracers.csvと同じ表記。一般人なら「一般女性」等 |
| confidence | 確度 | A | **A=本人公表 / B=メディア報道 / C=まとめサイト・噂** |
| source_url | 出典URL | https://… | 必ず入れる。決定版を名乗る生命線 |
| source_date | 出典の日付 | 2025-03-03 | 発表・報道された日 |
| checked | 最終確認日 | 2026-07-06 | |
| memo | 補足 | 師弟婚 | |

### 関係タイプ辞書（表記ゆれ禁止・この12種のみ）
| rel_type | 意味 | 逆方向 |
|---|---|---|
| 父 | fromの父がto | 子 |
| 母 | fromの母がto | 子 |
| 子 | fromの子がto | 父/母 |
| 兄 | | 弟/妹 |
| 姉 | | 弟/妹 |
| 弟 | | 兄/姉 |
| 妹 | | 兄/姉 |
| 配偶者 | 現在の婚姻 | 配偶者 |
| 元配偶者 | 離婚済み | 元配偶者 |
| 師匠 | fromの師匠がto | 弟子 |
| 弟子 | fromの弟子がto | 師匠 |
| 親族 | 従兄弟・叔父等はmemoに詳細 | 親族 |

### 運用ルール3箇条
1. **関係は片方向だけ書く**（峰→山田に「弟子」と書いたら、山田→峰の「師匠」行は作らない）。逆方向はサイト生成時にClaude Codeが自動で補う。二重管理は必ずズレる。
2. **confidence C（噂）はサイトに表示しない。** DBには記録して寝かせる。AかBに昇格した時だけ公開。
3. **source_urlが無い情報は入力禁止。** 出典なし＝存在しない扱い。
